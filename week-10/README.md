# Week 10 – Product Image Upload API

Extension of the Week 9 e-commerce backend with **multi-image upload**, **image validation**, and **automatic thumbnail generation**.

## What's New (Week 10)

| Feature | Details |
|---------|---------|
| Multiple images per product | Up to 5 images stored per product |
| Image validation | MIME type (JPEG/PNG/WebP) + max 5 MB per file |
| Auto-thumbnail | 300×300 px JPEG generated by Pillow, aspect-ratio preserved |
| Static file serving | Images served at `/uploads/products/` and `/uploads/thumbnails/` |
| New DB table | `product_image` — stores filename, URL, thumbnail, size, MIME, is_primary |

## New API Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `POST` | `/api/products/<id>/images` | Seller JWT | Upload 1-5 images (multipart/form-data, field: `images`) |
| `GET` | `/api/products/<id>/images` | Public | List all images for a product |
| `DELETE` | `/api/products/<id>/images/<img_id>` | Seller JWT | Delete an image + its thumbnail |
| `PATCH` | `/api/products/<id>/images/<img_id>/primary` | Seller JWT | Set an image as the cover photo |

Also: `GET /api/products/<id>` now includes an `images` array and `primary_image_url`.

## Static File URLs

After uploading, images are accessible at:
```
http://localhost:5000/uploads/products/<filename>       ← full original
http://localhost:5000/uploads/thumbnails/thumb_<filename>.jpg  ← 300×300 thumbnail
```

## Setup & Run

### 1. Install dependencies
```bash
cd week-10
pip install -r requirements.txt
```

### 2. Configure environment
Copy `.env.example` to `.env` and fill in your values (the defaults work with the docker-compose from week-9).

### 3. Start database (reuse week-9 docker-compose)
```bash
cd ../week-9
docker-compose up -d
cd ../week-10
```

### 4. Run the server
```bash
python main.py
```
Server starts at **http://localhost:5000**

## Image Upload Example (Postman)

1. Import `api.postman_collection.json`
2. Run **Register Seller** → **Login Seller** (token auto-saved)
3. Create a category, then a product
4. Run **Upload Images**:
   - Method: `POST`
   - URL: `http://localhost:5000/api/products/1/images`
   - Authorization: `Bearer <token>`
   - Body: `form-data`, key = `images`, type = `File`
   - Select up to 5 JPEG/PNG/WebP files

## Validation Rules

| Rule | Limit |
|------|-------|
| Allowed types | JPEG, PNG, WebP |
| Max file size | 5 MB per file |
| Max images per product | 5 |
| Type detection | Magic-number (actual file bytes), not extension |

## Thumbnail Details

- Generated by **Pillow** using `img.thumbnail((300, 300), LANCZOS)`
- Aspect ratio is preserved — result fits *within* 300×300, never stretched
- RGBA/palette PNGs are composited on a white background before JPEG save
- Saved as **JPEG quality=85, optimized=True**
